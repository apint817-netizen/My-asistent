# Документация по интеграции ИИ (Google Gemini & Local Proxy)

В проекте **My-asistent** реализована гибридная, надежная архитектура интеграции искусственного интеллекта. Сервис поддерживает два основных режима работы (провайдера), которые позволяют пользователю либо использовать бесплатные лимиты Google AI Studio, либо подключаться к локальным/кастомным моделям.

---

## 🏗 Архитектура интеграции

Интеграция разделена на клиентскую часть (React/Vite) и серверную часть (Vercel Serverless / Edge Functions). Это разделение критически важно для безопасности и обхода региональных блокировок.

### 1. Провайдеры ИИ

В настройках приложения (`src/components/AISettingsModal.jsx`) пользователь может выбрать один из двух путей:

#### A. Google Gemini (Основной, бесплатный режим)
- **Как работает:** Запросы отправляются **на сервер Vercel** (в папку `/api`), а Vercel уже делает запрос к серверам Google.
- **Безопасность ключа:** Серверный ключ `GOOGLE_API_KEY` скрыт в переменных окружения Vercel (`.env`). Ключ **никогда не попадает в браузер** (клиент-сайд).
- **Обход блокировок:** Поскольку серверы Vercel находятся в США/Европе, запросы к Google API всегда проходят успешно, даже если конечный пользователь находится в регионе, где Google Gemini недоступен (например, в России).
- **Важный нюанс (Fix):** Чтобы старые, невалидные ключи из памяти браузера пользователя (`localStorage`) не ломали запросы, интерфейс принудительно стирает клиентский ключ перед отправкой на сервер. Сервер берёт ключ напрямую из `process.env.GOOGLE_API_KEY`.

#### B. Локальный Прокси (Antigravity Tools / OpenAI формат)
- **Как работает:** Запросы отправляются напрямую из браузера на указанный пользователем локальный URL (например, `http://127.0.0.1:8045/v1`).
- **Формат:** Используется OpenAI-совместимый формат тела запроса.
- **Применение:** Идеально подходит для работы с локально запущенными моделями (LM Studio, Ollama) или приватными API-прокси. Вся логика генерации и расхода токенов переносится на сторону пользователя.

## 🛠 Роутинг и Фолбеки (Fallback System)

Главная "магия" стабильности ИИ находится в файлах `api/chat.js` и `api/validate.js`. 

### `api/chat.js` (Edge Runtime)
Обрабатывает основную генерацию текста, мультимодальные запросы (картинки, файлы) и команды ассистента.

**Цепочка отката (Fallback Chain):**
Google периодически меняет или отключает старые модели (например, `gemini-pro`, `gemini-1.5-flash`), что приводит к ошибкам "API key expired" на новых ключах. 
Поэтому мы реализовали систему "умного фолбека". Если одна модель не работает или перегружена (429 Too Many Requests), скрипт мгновенно пробует следующую:

1. Запрошенная модель (например, `gemini-2.0-flash`)
2. `gemini-2.5-flash`
3. `gemini-2.0-flash`
4. `gemini-2.0-flash-lite`

Если из-за ошибки 400, 404 или 429 модель отклоняет запрос, скрипт "проглатывает" ошибку и идёт дальше по массиву. **Ломка цепочки** происходит только при ошибках 401/403 (когда API-ключ действительно неверный).

### `api/validate.js` (Edge Runtime)
Используется для мгновенной валидации ввода пользователя при создании Задач и Наград. Предотвращает ввод случайного мусора (например, "asdasdasd").

- **Fail-safe (Безотказность):** Если у пользователя не настроен ключ, или API полностью лёг, скрипт **не блокирует UI**. Он отлавливает ошибку и всегда возвращает `{"valid": true}`, пропуская ввод. Пользовательский опыт важнее строгой валидации.

## ⚙️ Фронтенд: Парсинг и Управление стейтом (`AIAssistant.jsx`)

Умный компонент ассистента — это не просто чат, это **оркестратор приложения**. 
В `systemPrompt` (инструкциях для ИИ) жёстко задан набор текстовых тегов, которые ИИ должен использовать для взаимодействия с интерфейсом.

Примеры тегов:
- `[ADD_TASK: "Сделать зарядку" | 50]`
- `[ADD_CALENDAR_TASK: "Встреча" | 100 | "2026-03-01"]`
- `[ADD_REWARD: "Пицца" | 500]`

Когда ИИ возвращает текст, компонент (`generateAIResponse` в `AIAssistant.jsx`) "читает" ответ через регулярные выражения (`Regex`). 
Если находится тег, вызывается соответствующий метод из глобального управления состоянием (**Zustand**, `useStore.js`).

**Жизненный цикл команды:**
1. Пользователь: "Поставь мне задачу: прочитать книгу за 20 очков"
2. ИИ получает системный промпт (правила тегов) + сообщение пользователя.
3. ИИ генерирует ответ: "Конечно! [ADD_TASK: "Прочитать книгу" | 20]"
4. Наш фронтенд ловит ответ, вырезает тег из текста (чтобы он не отображался пользователю) и отправляет `"Прочитать книгу"` с ценностью `20` в `useStore.addTask()`.
5. UI обновляется мгновенно.

## 🚀 Деплой и безопасность

1. **Не держите ключи в коде.** В старых версиях был "вшит" демо-ключ, и он мгновенно исчерпывал квоту. Сейчас в коде нет ни одной строчки с `AIzaSy...`.
2. **Переменные Vercel.** При развёртывании сервиса на Vercel (или другой платформе), необходимо просто добавить `GOOGLE_API_KEY` в `Environment Variables`.
3. **Обновление ключа браузером.** Сайт хранит настройки в `localStorage`. Если вы решите перейти с Google API на кастомный, вы просто вбиваете данные в Настройках, браузер их запоминает, и `src/utils/geminiApi.js` начинает отправлять запросы в обход Vercel.
